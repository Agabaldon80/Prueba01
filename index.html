Pasos para usar git

  • Configurando git
  • git config --global user.name "mi nombre" configura nombre usuario git
  • git config --global user.email mi@email.com configura email git
  • Iniciando git
  • git init - inicia repositorio git en nuestro proyecto, se hace 1 sola vez
  • Trabajando con git
  • git add . - guardamos todos los archivos creados o modificados con . o uno por uno con su nombre
  • git commit -m “titulo del commit” - no queda nada para commit
  • git status - estado y saber si tengo que guardar o hacer commit
  • git log - muestra los commit realizados
  • Trabajando ramas
  • git branch . - Con este comando veo el listado de ramas creadas.
  • git branch nombre-de-rama - Para crear una rama nueva.
  • git checkout nombre-de-rama - Con este comando me muevo a la rama seleccionada.
  • git merge nombre-de-rama - !IMPORTANTE¡ Debo estar parado en la rama en la cual deseo traer los cambios de la rama externa. (es decir si quiero traerme los cambios de la rama2 a main , debo estar parado en main)
  • git branch -m nombre-actual nuevo-nombre - Con este comando renombro la rama actual por el nombre de la nueva rama.
  • git branch -d nombre-de-rama - Comando para borrar la rama local en git.
  • git push origin --delete nombre-de-rama - Comando para borrar la rama online en github.
  • TRABAJANDO DE FORMA REMOTA CON GITHUB
  • git remote add origin url-repositorio - Conecta mi repo local al repo online de github
  • git remote -v - Me muestra el url del repo online en github.
  • git push -u origin main - Empuja los cambios de mi repo local al repo de github. !ATENCION¡ el origin "main" puede ser reemplazado por el nombre de la rama que quieren pushear "git push -u origin otra-rama".
  • git remote rm origin - Este comando borra la conexion del repo local con el repo online de github. (Hecho esto hacer el paso de git remote add origin para conectar el nuevo repositorio online)
  • git branch -m nombre-actual nuevo-nombre - Con este comando renombro la rama actual por el nombre de la nueva rama.
  • git clone url-del-repo-online - Crea un clon del repo de github en mi repo local.
  • git pull - Trae todos los cambios que existan en la rama online de github.
  • git fetch - Trae todos los cambios que existan en todas las ramas online de github.
